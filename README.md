# Ollama Agent - 智能对话助手

一个功能完整的 Ollama 大模型调用系统，支持本地 Ollama 服务和外部 API 调用，具备视觉分析、音频处理、多媒体交互等高级功能。

## ✨ 主要功能

### 🤖 核心功能
- **多模型支持**: 同时支持本地 Ollama 和外部 API 调用
- **智能切换**: 根据问题类型自动选择合适的模型（文本/视觉/音频）
- **流式响应**: 实时流式输出，支持逐字显示
- **对话管理**: 完整的对话历史记录和管理
- **思考链**: 可控制的思考链显示功能

### 👁️ 视觉功能
- **实时摄像头**: 支持摄像头实时预览和拍照
- **图片分析**: 支持常见图片格式（jpg, png, gif）分析
- **视觉模型**: 自动检测和使用 qwen2.5-vl 等视觉模型
- **多图上传**: 支持批量图片文件上传和分析

### 🎵 音频功能
- **语音录制**: 实时语音录制和上传
- **音频处理**: 支持 mp3, wav 等常见音频格式
- **智能识别**: 自动识别音频相关问题并使用相应模型

### 📁 文件管理
- **安全上传**: 支持多种文件格式上传
- **权限控制**: 可配置允许访问的目录和文件类型
- **拖拽上传**: 支持拖拽文件到聊天界面
- **文件预览**: 上传文件列表管理和预览

### 🎛️ 系统监控
- **实时监控**: 命令行实时监控系统状态
- **连接状态**: 实时显示 Ollama 和 Agent 连接状态
- **资源监控**: CPU、内存、磁盘使用情况
- **模型统计**: 可用模型数量和类型统计

### 💻 代码解释器
- **安全沙盒**: 隔离的代码执行环境，确保系统安全
- **Python 支持**: 支持 Python 代码编写和执行
- **数据处理**: 可运行代码处理数据、生成图表和文件
- **实时输出**: 流式输出执行结果和错误信息
- **文件管理**: 支持生成文件的下载和管理

### 🔄 自动化脚本
- **智能启动**: 集成所有启动脚本的自动化启动器
- **环境检查**: 自动检测系统环境和依赖
- **全面测试**: 执行完整的项目测试流程
- **条件启动**: 仅当测试通过后自动启动应用
- **错误诊断**: 详细的失败报告和错误分析

## 🚀 快速开始

### 环境要求
- Python 3.8+
- Ollama 服务（可选）
- 现代浏览器（Chrome, Firefox, Safari, Edge）

### 安装步骤

1. **克隆或下载项目**
   ```bash
   # 如果从Git下载，解压到任意目录
   cd ollama_agent
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **启动系统**

   **推荐方式 - 自动化启动器（支持所有平台）:**
   ```bash
   python launcher.py
   ```

   **传统方式:**
   
   **Linux/Mac:**
   ```bash
   chmod +x start.sh
   ./start.sh
   ```

   **Windows:**
   ```cmd
   start.bat
   ```

4. **访问界面**
   
   - Web 界面: http://localhost:5000
   - 监控界面: 启动监控选项

### 手动启动

如果启动脚本有问题，可以手动启动：

```bash
# 启动Web服务
python app.py

# 启动监控（另开终端）
python monitor.py
```

## 📖 使用指南

### 基本操作

1. **选择模型**
   - 在左侧边栏选择可用的模型
   - 系统会自动标识模型类型（文本/视觉/音频）

2. **开始对话**
   - 在底部输入框输入消息
   - 按 Enter 发送，Shift+Enter 换行

3. **使用摄像头**
   - 点击摄像头按钮开启预览
   - 点击拍照按钮捕获图像
   - 图像会自动添加到对话中

4. **上传文件**
   - 点击上传按钮或拖拽文件到聊天区域
   - 支持图片、文档、音频、视频等多种格式

5. **语音输入**
   - 点击语音输入按钮开始录音
   - 再次点击或按停止按钮结束录音

### 高级功能

1. **思考链控制**
   - 开启/关闭思考链显示
   - 思考过程会显示在回复中

2. **设置配置**
   - 点击设置按钮打开配置面板
   - 配置外部 API 密钥和地址
   - 设置文件访问权限

3. **系统监控**
   - 启动监控查看实时状态
   - 监控系统资源和连接状态

### 模型配置

#### 本地 Ollama
确保 Ollama 服务正在运行：
```bash
ollama serve
```

安装模型：
```bash
ollama pull llama3.1
ollama pull qwen2.5-vl  # 视觉模型
ollama pull whisper     # 音频模型
```

#### 外部 API
在设置中配置：
- API Key: 您的 API 密钥
- API Base URL: API 服务地址

支持的 API：
- OpenAI API
- 其他兼容的 API 服务

## � 代码解释器使用指南

### 功能介绍
代码解释器是一个安全的沙盒执行环境，允许您在对话中编写和执行 Python 代码，用于数据处理、图表生成和文件操作。

### 使用方法

1. **打开代码解释器**
   - 在聊天界面中，点击代码解释器按钮
   - 代码编辑器面板将出现在聊天区域下方

2. **编写和执行代码**
   ```python
   # 示例：数据处理和图表生成
   import numpy as np
   import matplotlib.pyplot as plt
   
   # 生成示例数据
   x = np.linspace(0, 10, 100)
   y = np.sin(x)
   
   # 创建图表
   plt.figure(figsize=(8, 4))
   plt.plot(x, y)
   plt.title("正弦波示例")
   plt.xlabel("X轴")
   plt.ylabel("Y轴")
   plt.grid(True)
   
   # 保存图表
   plt.savefig("sine_wave.png")
   print("图表已保存为 sine_wave.png")
   ```

3. **查看执行结果**
   - 代码执行结果会实时显示在输出区域
   - 支持标准输出、错误信息和文件下载链接

4. **安全限制**
   - 代码在隔离的沙盒环境中执行
   - 限制访问系统敏感资源
   - 执行时间限制为30秒
   - 输出大小限制为10MB

### 支持的功能
- **数据处理**: NumPy, Pandas 等数据分析库
- **图表生成**: Matplotlib, Plotly 等可视化库
- **文件操作**: 读取、写入、生成文件
- **数学计算**: 复杂的数学运算和统计分析

## 🔄 自动化脚本使用指南

### 功能概述
自动化启动器 `launcher.py` 整合了所有启动和测试功能，提供智能化的项目启动流程。

### 启动选项

```bash
# 完整流程（推荐）
python launcher.py

# 仅检查环境
python launcher.py --check-only

# 仅运行测试
python launcher.py --test-only

# 快速启动（跳过测试）
python launcher.py --quick
```

### 执行流程
1. **环境检查** - 验证 Python 版本、依赖包、Ollama 服务
2. **应用启动** - 启动 Flask 应用并检查运行状态
3. **集成测试** - 执行完整的 API 和功能测试
4. **条件启动** - 仅当测试全部通过后进入启动菜单

### 交互式菜单
测试通过后显示启动选项：
- **1. 启动 Web 应用** - 启动主应用服务
- **2. 同时启动监控工具** - 启动 Web 应用和系统监控
- **3. 仅启动监控工具** - 启动系统监控
- **4. 清理缓存和冗余文件** - 清除测试产生的缓存和临时文件
- **5. 退出** - 退出启动器

### 错误处理
- **详细诊断**: 测试失败时生成完整的错误报告
- **环境信息**: 包含系统配置、依赖版本等详细信息
- **堆栈跟踪**: 完整的错误调用链分析

## � 故障排除

### 常见问题

1. **Ollama 连接失败**
   ```bash
   # 检查 Ollama 服务状态
   curl http://localhost:11434/api/tags
   
   # 启动 Ollama 服务
   ollama serve
   ```

2. **端口被占用**
   ```bash
   # 查看端口占用
   lsof -i :5000
   
   # 杀死占用进程
   kill -9 <PID>
   ```

3. **摄像头无法访问**
   - 检查浏览器权限设置
   - 确保没有其他应用占用摄像头
   - 尝试刷新页面

4. **文件上传失败**
   - 检查文件大小（默认限制 16MB）
   - 确认文件类型是否支持
   - 检查目录权限

5. **音频录制失败**
   - 检查麦克风权限
   - 确保浏览器支持 WebRTC
   - 检查系统音频设置

### 性能优化

1. **内存使用**
   - 关闭不需要的模型
   - 定期清理对话历史
   - 监控系统资源使用

2. **响应速度**
   - 选择较小的模型
   - 关闭思考链功能
   - 优化网络连接

3. **并发处理**
   - 避免同时处理多个大文件
   - 限制同时进行的对话数量

## 🔒 安全说明

1. **文件访问控制**
   - 默认只允许访问上传目录
   - 可配置允许访问的特定目录
   - 文件大小和类型限制

2. **网络安全**
   - 服务默认绑定到所有接口
   - 生产环境建议配置防火墙
   - 可考虑使用 HTTPS

3. **数据隐私**
   - 对话历史存储在本地
   - 不会主动上传数据到外部服务器
   - 外部 API 调用需要用户授权
